Идея синхронизации нод состои в том, чтобы иметь какуюто Мастер ноду и много
обычных нод. Таким образом при запуске/по указанию пользователя/еще как-то
нода связывается с Мастером(адрес берётся из конфигов/броадкастится) и отдаёт
ему списки объектов бд(врачи, пациенты, обследования, страховые фирмы). На что
в ответ Мастер высылает недостающие клиенту объекты бд, а также запрашивает
те, которых нет у него. После того как клиент выслал запрошеные Мастером ноды
синхронизация считается законченой.

Синхронизирование нод должно проходить по следующей схеме:

	1. Клиент связывается с Мастер-нодой. (Конфиг + бродкаст)
	2. Клиент шлёт список всех имеющихся объектов БД.
	3. Мастер-нода отдаёт недостающие(новые для ноды) объекты БД, а также
	запрашивает недостающие ему.
	4. Клиент отдаёт Мастеру все запрошеные объекты БД.
